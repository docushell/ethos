# Fixture Corpus and Contribution Guide

Fixtures are the project's evidence. Heuristic changes without fixtures are rejected in review
(fixtures-first, PRD §14). Failures with fixtures become regression tests.

## Layout

```text
fixtures/
  public/      # real-world PDFs that are redistributable (public domain or licensed)
  synthetic/   # generated PDFs we own outright (preferred for targeted cases)
  security/    # hidden/off-page/low-contrast text, annotations, actions, scripts, links
  failure/     # corrupt, encrypted, password, image-only, oversized, rotated
```

Each fixture is a directory: the PDF, a `fixture.json` (provenance, license, what it
exercises, expected codes), and goldens added per pipeline stage as lanes land
(`extraction.json`, `layout.json`, `tables.json`, `chunks.jsonl`, …).

## Contribution rules

1. **License attestation is mandatory.** Only public-domain, synthetic, or
   redistribution-licensed PDFs. State provenance and license in `fixture.json`. We do not
   accept "found it somewhere".
2. **No PII.** Real documents must be scrubbed or rejected.
3. **Synthetic preferred** for targeted behaviors (one quirk per fixture where possible).
4. **Name by behavior**, not source: `ligature-fi-embedded-font/`, `table-borderless-2col/`,
   `hidden-text-white-on-white/`.
5. **Declare the subset** in `fixture.json`: `born_digital`, `security`, `failure`,
   `fonts`, `rotation`, `tables`, `multi_column`, …
6. Goldens are generated by maintainers under the pinned deterministic profile and reviewed as
   diffs; contributors don't need to produce them.

## Relationship to Gate Zero

The Gate Zero corpus (`benchmarks/gate-zero/manifest.json`) is a frozen, hash-pinned snapshot
selected by the corpus owner — fixtures contributed here feed later corpus revisions, but the
frozen manifest never changes after kickoff (changes void the measurement, PRD §1.3).

Ethos' own fixtures are never presented as a neutral benchmark suite (PRD §11.2).
