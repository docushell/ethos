# Real OpenDataLoader Grounding Fixture

This fixture pins real `opendataloader-pdf` JSON output, not the synthetic
OpenDataLoader-style subset used in `examples/verify/`.

## Contents

- `source.pdf`: upstream OpenDataLoader sample PDF, `samples/pdf/lorem.pdf`
- `opendataloader-output.json`: generated by `opendataloader-pdf` 2.4.7
- `citations.json`: AI-answer-style citation claims grounded by the output JSON
- `ungrounded_citations.json`: citation claims that must not fully ground against the output JSON
- `expected.verification_report.json`: Ethos verification report golden
- `expected.ungrounded.verification_report.json`: Ethos negative-path report golden
- `manifest.json`: source, generator, and hash provenance

## Reproduction

```bash
opendataloader-pdf \
  --quiet \
  --format json \
  --image-output off \
  --threads 1 \
  --output-dir /tmp/ethos-real-odl \
  source.pdf
```

The pinned command is single-threaded and disables image extraction so this fixture stays focused
on text and citation grounding. The OpenDataLoader JSON shape does not include parser version or
page dimensions, so the Ethos adapter reports parser version as `unknown`, derives page extents
from observed bounding boxes, keeps coordinate origin as `unknown`, and does not declare table
capability for real ODL output yet.
